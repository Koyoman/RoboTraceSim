# Robotrace Sim v0.5 — Comparação com dados reais

## Objetivo

Adicionar uma camada de calibração para reaproveitar dados de robô real e medir o quanto a simulação se aproxima do comportamento observado.

## Implementado

### Importação de log real

- Novo módulo `src/calibration.rs`.
- Função `import_real_log` para CSV real.
- Detecção de colunas de tempo:
  - `t_us`
  - `time_us`
  - `timestamp_us`
  - `t_s`
  - `time_s`
  - `t_ms`
  - `time_ms`
- Detecção de colunas de pose, velocidade, linha e sensores:
  - `x_m`, `y_m`, `yaw_rad`
  - `vx_body_m_s`, `speed_m_s`, `v_m_s`
  - `yaw_rate_rad_s`
  - `line_position_m`, `line_error_m`
  - `sensor_00_adc`, `sensor_01_adc`, etc.
- Comando CLI:

```bash
robotrace-sim import-log real.csv --output real_normalized.csv
```

### Comparação simulação vs real

- Execução da simulação em memória usando `SimulationSession`.
- Alinhamento temporal por interpolação linear entre amostras simuladas.
- Comparação no tempo das amostras reais.
- Saídas:
  - CSV detalhado por amostra.
  - TXT resumido.
  - impressão no terminal.

Comando:

```bash
robotrace-sim compare projeto.rtsim --real real.csv --output comparacao.csv --report comparacao.txt
```

### Métricas

Foram adicionadas métricas com `count`, `mean_abs`, `rms` e `max_abs` para:

- Erro de trajetória em metros.
- Erro de yaw em radianos.
- Erro de velocidade em m/s.
- Erro de yaw rate em rad/s.
- Erro de posição da linha em metros.
- Erro de erro de linha/controlador em metros.
- Erro de sensores em ADC.

Também foi adicionado um score normalizado para comparação entre candidatos de calibração.

### Ajuste de parâmetros

- Comando CLI:

```bash
robotrace-sim tune projeto.rtsim --real real.csv --output ajuste.json
```

- Busca grossa determinística sobre:
  - `tire.mu_longitudinal`
  - escala do torque de stall dos motores
  - escala da corrente de stall dos motores
- O resultado é salvo em JSON sem sobrescrever automaticamente o projeto.

### Interface gráfica

- Nova aba `Calibração v0.5`.
- Campos para log real, CSV de comparação, relatório TXT e JSON de ajuste.
- Botões:
  - Importar/normalizar log real.
  - Comparar simulação vs real.
  - Ajustar parâmetros.
- Exibição das últimas métricas principais.

## Limitações atuais

- O parser de CSV é próprio e simples; funciona para logs comuns, mas não substitui um parser CSV completo.
- O ajuste de parâmetros ainda é uma busca grossa fixa, não uma otimização avançada.
- O ajuste não aplica automaticamente os valores ao `robot.json` para evitar sobrescrita acidental.
- O ambiente de montagem desta entrega não possui `cargo`/`rustc`, então a validação foi feita por revisão estática e validação dos JSONs de exemplo.
