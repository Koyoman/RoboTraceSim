# Versão 0.6 — Track Geometry Foundation

Esta etapa substitui a pista editada como pontos soltos por uma pista paramétrica fechada em `rtsim-track-v2`.

## Implementado

- Novo módulo `rtsim_track`.
- Segmentos geométricos ordenados: reta e arco circular.
- Convenção de arco: `sweep_deg > 0` curva para esquerda, `sweep_deg < 0` curva para direita.
- Construção determinística da linha central a partir de origem + segmentos.
- Cache de `centerline` derivado para manter compatibilidade com sensores/simulação atuais.
- Cálculo de comprimento total e erro de fechamento `dx`, `dy`, `dθ`.
- Validador com severidade `Error`, `Warning` e `Info`.
- Perfis de regras com modos `strict`, `warning` e `free`.
- Overrides editáveis para largura de linha, comprimento máximo, raio mínimo e demais regras principais.
- Validações prioritárias: fechamento, segmentos válidos, IDs únicos, raio mínimo, comprimento máximo, mudanças de curvatura, borda da área, largada/chegada e cruzamentos amostrados.
- Editor egui com tabela paramétrica de segmentos.
- Canvas com fundo preto, linha branca, grade, origem, direção, largada, chegada e corner markers.
- Botões para adicionar reta, adicionar arco à esquerda, adicionar arco à direita, remover, auto close por reta final e otimizar última reta.
- Exportação JSON no novo formato `rtsim-track-v2`.
- Leitura do formato legado `rtsim-track-v1` preservada; o editor oferece criação de uma pista v2 fechada.

## Decisão arquitetural

A geometria paramétrica é a fonte oficial. A polyline usada pela simulação atual é apenas cache derivado, até que os próximos passos implementem raster óptico, mapa de atrito e índice espacial.
