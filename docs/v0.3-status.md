# Robotrace Sim v0.3 — Downforce/sucção

Esta versão evolui a base v0.2 com um módulo explícito de força normal, permitindo simular aumento de aderência por ventoinhas ou sucção.

## Implementado

- **`NormalForceModel` configurável**
  - Novo arquivo `src/normal_force.rs`.
  - Modelo padrão compatível: `NoDownforce`.
  - Modelos adicionados: `ConstantDownforce`, `FanDownforce`, `SuctionDownforce`, `MeasuredDownforceCurve` e base para `ClosedLoopDownforce`.

- **FanDownforce**
  - Suporte a múltiplos fans por robô.
  - Cada fan possui posição no chassi, força máxima, corrente máxima, tensão nominal, constante de resposta e curva medida `force_curve`/`thrust_curve`.
  - Força por curva medida PWM → N ou, se a curva não existir, aproximação `F = Fmax * pwm²`.
  - A posição do fan afeta a distribuição de normal nas rodas.

- **SuctionDownforce**
  - Suporte a câmara de sucção com área efetiva, pressão máxima, vazamento, corrente máxima e tempo de resposta.
  - Modelo base: `F_suction = ΔP * A_effective`, com alternativa por curva medida PWM → N.
  - A força é aplicada em `position_mm`, permitindo deslocar o centro de sucção no chassi.

- **Distribuição de normal nas rodas**
  - O simulador agora calcula:
    - `normal_front_left_n`
    - `normal_front_right_n`
    - `normal_rear_left_n`
    - `normal_rear_right_n`
  - A distribuição usa uma aproximação bilinear sobre o retângulo de apoio do robô, considerando centro de massa e posição dos fans/sucção.
  - As rodas esquerda/direita continuam sendo agregadas para o diferencial, mas agora recebem normal derivada das quatro rodas.

- **Efeito da normal sobre atrito**
  - O limite longitudinal continua usando `Fmax = μ * N`.
  - Como `N` agora inclui peso + fan/sucção distribuídos, o downforce aumenta o limite de tração, frenagem e curva.
  - O limite lateral usa a normal total calculada pelo módulo.

- **PWM de fan/sucção**
  - O controlador possui `downforce_pwm`.
  - O `ControllerOutput` agora exporta `pwm_downforce`.
  - O módulo de normal usa esse PWM para calcular força e consumo.

- **Curvas medidas**
  - Aceita curvas no JSON como array de pontos `[[pwm, force_n], ...]`.
  - Para fans: `force_curve`, `thrust_curve` ou `measured_curve`.
  - Para sucção/downforce global: `force_curve` ou `measured_curve`.

- **Telemetria e replay v3**
  - CSV registra PWM de downforce, normal por roda, força extra, força de fan, força de sucção e corrente do sistema de downforce.
  - Replay binário passou para magic/version `RTSRPL03`/`3` e inclui os novos campos.

## Exemplos

Fan/downforce:

```bash
cargo run --release -- run examples/basic/projeto.rtsim --headless --duration 10s
```

Sucção:

```bash
cargo run --release -- run examples/basic/projeto_suction.rtsim --headless --duration 10s
```

Exportar replay:

```bash
cargo run --release -- export examples/basic/resultado.rtlog --format csv --output examples/basic/resultado_exportado.csv
```

## Observação de validação

O ambiente usado para gerar este pacote não possui `cargo`/`rustc` instalado. Portanto, não foi possível executar `cargo test` localmente aqui. A revisão feita foi estática, e os JSONs de exemplo foram validados com `python -m json.tool`.
