# Robotrace Sim v0.4 — Interface única

Esta versão adiciona a primeira interface gráfica integrada em `egui/eframe`, preservando a arquitetura definida na especificação: a UI configura, visualiza e aciona comandos, mas o núcleo de simulação continua funcionando sozinho por terminal.

## Implementado

- **App única**
  - O executável agora abre a interface gráfica ao rodar sem argumentos.
  - Também é possível abrir explicitamente com `robotrace-sim ui`, `robotrace-sim gui` ou `robotrace-sim app`.
  - Os comandos CLI `run`, `benchmark` e `export` continuam disponíveis.

- **Home**
  - Tela inicial para abrir projeto `.rtsim`, usar o exemplo básico, criar um projeto novo em memória e salvar tudo.
  - Exibe resumo do projeto, robô, pista, período físico, duração, sensores e módulo de downforce/normal.

- **Editor de pista**
  - Edição de nome, modelo, largura da linha, refletâncias e atrito da superfície.
  - Canvas com `egui::Painter` para visualizar a pista vetorial.
  - Seleção e arrasto de pontos da `centerline_m`.
  - Tabela editável de pontos x/y e botões para adicionar/remover pontos.
  - Salvamento do `track.json` junto com o projeto.

- **Editor de robô**
  - Edição de chassi, rodas/transmissão, pneu, motores, driver, bateria, sensores, encoder, gyro, PID e downforce/sucção.
  - Suporte visual para trocar modelos como `NoDownforce`, `FanDownforce`, `SuctionDownforce` e `MeasuredDownforceCurve`.
  - Edição da lista de fans com posição, força máxima e corrente máxima.
  - Salvamento do `robot.json` junto com o projeto.

- **Simulador visual**
  - Novo `SimulationSession` incremental em `src/sim.rs`.
  - Reutiliza o mesmo núcleo determinístico fixed-step do comando `run`.
  - Permite `Reset`, `Play/Pause`, `Step`, ajuste de duração e quantidade de passos físicos por frame.
  - Desenha pista, robô, orientação e telemetria instantânea.
  - Pode gerar replay/CSV pela simulação headless diretamente pela interface.

- **Replay viewer**
  - Carrega arquivos `.rtlog` v3.
  - Novo leitor público `load_replay_samples` em `src/replay.rs`.
  - Slider de amostra para navegar no tempo.
  - Desenho de trajetória e pose atual.
  - Painel com telemetria: pose, velocidade, PWM, bateria, normal por roda, downforce, slip e encoders.
  - Exportação do replay para CSV pela interface.

- **Persistência sem `serde`**
  - A v0.4 mantém o parser JSON próprio da base anterior.
  - A UI escreve `projeto.rtsim`, `robot.json` e `track.json` usando serialização manual compatível com os schemas atuais.

## Como abrir a interface

```bash
cargo run --release
```

ou:

```bash
cargo run --release -- ui
```

## Como rodar somente o núcleo CLI

```bash
cargo run --release -- run examples/basic/projeto.rtsim --headless --duration 10s
cargo run --release -- benchmark examples/basic/projeto.rtsim --duration 10s --physics-dt-us 500
cargo run --release -- export examples/basic/resultado.rtlog --format csv --output examples/basic/resultado_exportado.csv
```

## Observação de validação

O ambiente usado para gerar este pacote não possui `cargo`/`rustc` instalado. Portanto, não foi possível executar `cargo check` ou `cargo test` localmente aqui. A revisão feita foi estática, e os JSONs de exemplo existentes foram validados com `python -m json.tool`.
