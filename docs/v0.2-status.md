# Robotrace Sim v0.2 — Realismo básico

Esta versão expande a base headless determinística da v0.1 com os primeiros módulos de realismo físico e de telemetria.

## Implementado

- **Slip ratio**
  - Novo `SlipRatioWheel`.
  - O estado físico agora armazena velocidade angular e ângulo acumulado das rodas.
  - Quando a força exigida excede `mu * N`, a roda pode acumular diferença entre velocidade periférica e velocidade do solo.
  - O CSV registra `slip_left`, `slip_right`, `wheel_surface_speed_left_m_s` e `wheel_surface_speed_right_m_s`.

- **Bateria com queda de tensão**
  - Novo `VoltageSagBattery`.
  - Tensão terminal calculada por tensão em circuito aberto menos `I * R_internal`.
  - Estado de carga (`soc`) é atualizado internamente a partir da corrente consumida.
  - O CSV registra `battery_voltage_v` e `battery_current_a`.

- **Driver PWM com brake/coast**
  - `DcMotorSimple` agora respeita `driver.mode`.
  - `coast`: PWM zero deixa o motor livre.
  - `brake`: PWM zero aplica torque contrário à rotação por back-EMF aproximada.
  - PWM tem quantização por `pwm_resolution_bits`, `command_deadband`, `voltage_drop_v` e limite de corrente.

- **Encoder quantizado**
  - Novo `QuantizedEncoder`.
  - Gera ticks inteiros cumulativos e velocidade estimada por delta de ticks.
  - Configurável por `ticks_per_rev`, `invert_left` e `invert_right`.

- **Gyro com ruído**
  - Novo `NoisyGyro`.
  - Inclui bias, saturação e ruído gaussiano determinístico por seed.
  - O CSV registra `gyro_yaw_rate_rad_s` e `gyro_bias_rad_s`.

- **Sensor com ADC e ruído**
  - `SimpleLineSensor` agora funciona como `NoisyAdcSensor` configurável.
  - Inclui ganho, offset, ruído de refletância, ruído em LSB e quantização ADC.

- **Replay binário**
  - Novo `BinaryReplayLogger` com formato `.rtlog`.
  - `run` aceita `--replay resultado.rtlog`.
  - `--output arquivo.rtlog` também seleciona replay automaticamente.
  - Novo comando:
    ```bash
    robotrace-sim export resultado.rtlog --format csv --output resultado_exportado.csv
    ```

## Observação de validação

O ambiente usado para gerar este pacote não possui `cargo`/`rustc` instalado. Portanto, a revisão feita aqui foi estática. Recomenda-se rodar localmente:

```bash
cargo fmt
cargo test
cargo run --release -- run examples/basic/projeto.rtsim --headless --duration 10s
cargo run --release -- export examples/basic/resultado.rtlog --format csv --output examples/basic/resultado_exportado.csv
```
